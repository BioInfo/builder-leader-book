# Canon — reserved

This directory normally holds the canonical spine of the book:

- `thesis.md` — the single-paragraph pitch, one-sentence thesis, three-bullet argument skeleton.
- `outline.md` — chapter list with one-paragraph synopsis per chapter.
- `audience.md` — who the book is written for, what they take away.
- `book-voice.md` — book-specific extensions to the author's global voice canon.
- `voices-to-track.md` — six tiers of voices in the topic landscape (anchors, allies, contrarians, peers, etc.) plus the rotation rules across them.
- `book-craft-notes.md` — craft and structural decisions made during drafting.
- `research-rules.md` — rules for what counts as a citation, what gets sourced, what gets cut.

**These files are not in the public repo.** The canonical spine is the secret sauce. It defines what the book is, what it argues, and how it sounds. Publishing it would let any reader reconstruct the book's structure and reasoning without buying it.

What you can see in this repo:

- The **shape** of the canon — this README enumerates the file types and what role each plays.
- The **enforcement** of the canon — every skill in `.claude/skills/` reads canon files at session start. The skill SKILL.md files describe what they look for.
- The **output** of canon-grounded work — see the preface in `chapters/_sample/`, the sample research and review-work artifacts under `_samples/` directories.

The full canon is held in the author's private working repo. Reading the book gives you the argument; this repo's structure gives you the framework.
