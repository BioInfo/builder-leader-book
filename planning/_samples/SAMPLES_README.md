# Sample planning artifacts

The planning directory in the production repo holds session logs, the rolling next-session agenda, and decision records. This `_samples/` folder shows what one of those looks like in practice.

## What's here

- `session-during development-stage-6-closeout-and-ship.md` — the real session log from the day the manuscript shipped. Documents what edits landed, which author-judgment questions resolved, the ship-time citation queue, and the `book-export` ship mode output. Personal filesystem paths were stripped; the content is otherwise verbatim.
- `next-session-agenda-template.md` — a clean template for the "first moves of next session" file format. Used at the top of every session to orient.

## Why share session logs

Two reasons.

First, they make the production legible. Most published nonfiction does not show readers the day-to-day mechanics: which sentences got rewritten, which citations got cut, which questions the author had to make a call on. The session log is the audit trail for those decisions.

Second, they're transferable. The format is reusable: date, slug, what-this-session-was, edits-applied, status-changes, what's-still-open. Drop the format into your own writing project and you get a low-overhead way to remember what you did between sessions, especially across long gaps.

## What's NOT here

The full `planning/sessions/` directory in the production repo holds dozens of session logs spanning the whole drafting arc. Most reference internal context (other projects, personal stakeholder names, in-flight decisions) and are kept private. The single ship-day log is included as the most useful representative — it captures the close-out gate that every chapter passed through.
