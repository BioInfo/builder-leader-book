# Field Guide 07 — Four exercises for weeks two through six

Companion to Chapter 7, *Building Your Personal Harness.*

By week two you've installed Claude Code, written a skill, and written a guardrail. The next four weeks turn that starter rig into a real harness you operate. Four exercises, one per week. The work is yours; the artifacts are yours.

## Week 1 — Notice what you do twice

Keep a sticky note on your monitor or a running file labeled `did-this-twice.md`. Every time you find yourself repeating a multi-step task by hand — formatting a report, summarizing a meeting, prepping a brief, replying to a class of email — write it down. One line each.

At the end of the week, read the list. Pick the three highest-frequency items. Each one is a candidate skill.

Write at least one of them as a real skill markdown file. Use it the next time it comes up.

Success criterion: the list has at least ten entries, and at least one became a working skill.

## Week 2 — Two-file memory without overengineering

The minimal memory architecture is two files. One holds project context, the other holds learnings about you and how you work. Both live in your `~/.claude/` directory or a project-specific equivalent.

Resist the urge to build a memory system. Resist any blog post that tells you to use a vector database for personal memory in week three. Two markdown files. Update them when you notice something worth remembering.

Success criterion: you have two memory files, you've updated each at least three times, and your sessions have started to feel like they remember you.

## Week 3 — Spawn one agent

An agent in this context is a long-running directive: a markdown spec, a tool, and a starting prompt that lets Claude Code work without you watching every step. The simplest first agent is a research summarizer or a status reporter — something that takes a directory or a list as input and produces a structured output.

Write the agent. Run it. Read what it produces. Adjust the spec until the output is something you'd actually use.

Success criterion: you have one named agent, you've run it at least three times, and you've tightened the spec each time.

## Week 4 — Ship one real piece of work

By the end of week four, ship one piece of work through your harness. Not a tutorial. Not a sandbox. A real deliverable for a real audience: a memo, a report, a draft strategy, a code review, a competitive analysis.

The work has to land somewhere outside your machine. Send it. Post it. Commit it. Hand it to someone whose opinion of your work you care about.

Success criterion: the work shipped, and at least one downstream person responded to it as if it were normal output from you.

## After

At week six you have a harness with skills, memory, an agent, and one shipped artifact. That's enough to keep going for several months without architectural change. The next chapter is what happens when a second person joins you.
