# Field Guide 04 — Inventory and first addition

Companion to Chapter 4, *The Harness and What's In It.*

The chapter names the six components of an operator harness: skills, memory, agents, hooks, MCP servers, and sensors-and-gates. The chapter also makes the case that the harness is eating SaaS — that the work being done by your subscription stack is increasingly available as harness behavior on your own machine.

Block thirty minutes. You'll do an inventory and pick a first addition for next week.

## Part 1 — The six-component inventory

For each component, write what you already have, even informally. Most readers will discover they have more than they thought.

- **Skills.** Repeatable instructions you've written for a tool to follow. Even a Cursor rule file or a long ChatGPT system prompt counts.
- **Memory.** Files or systems that hold context across sessions. Notes apps, project READMEs, even your inbox.
- **Agents.** Anything that runs without you watching every step. Auto-formatters, scheduled scripts, Zapier flows.
- **Hooks.** Triggers that run on events. Git hooks, IFTTT, calendar reminders that fire actions.
- **MCP servers.** Anything that connects an AI tool to a data source you control. Most readers won't have one yet.
- **Sensors and gates.** Checks that fire on data crossing a threshold. Monitoring alerts, spam filters, budget warnings.

The point of the inventory is not to score yourself. It's to notice that the harness pattern already runs in fragments inside your existing tools.

## Part 2 — The harness-eats-SaaS walk

Open the list of SaaS subscriptions your team or company pays for. Any subset will do — start with the ones in your direct portfolio.

For each tool, ask:

- What is this tool actually doing for us?
- Could a harness, directed by a senior operator, do the same job from raw inputs?
- If yes, what's blocking us from running that experiment?

You're not looking to cancel subscriptions. You're calibrating where the value of the harness pattern starts compounding against your own stack.

## Part 3 — Name a first addition

Pick one component from Part 1 that you don't currently have, and pick a specific first addition you'll make next week.

The most common first additions for executives are:

- A skill that codifies a repetitive prompt you keep retyping.
- A memory file that holds the context of a current project so AI tools start each session inside it.
- A hook that runs a check or a summary on a recurring trigger.

Write the name of the component and the specific thing. One line each. That line is your week-one move.
